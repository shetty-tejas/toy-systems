# Toy Systems

A small workspace for learning-oriented systems projects.

Each project here is meant to be a real implementation of one core idea, kept deliberately small enough to understand end to end. The goal is not production-readiness or feature parity with the original system; the goal is to learn by building.

## Built So Far

- [Ledger](ledger/README.md) — a Kafka-inspired segmented commit log written in Go.

## What These Projects Are For

- Understanding systems by implementing them
- Experimenting with storage layouts, APIs, and tradeoffs
- Using documentation, research, tests, and reviews to sharpen the design
- Keeping scope narrow and honest

## What They Are Not For

- Pretending to be production infrastructure
- Copying every feature of the original system
- Adding complexity that does not help the learning goal

## How AI Is Used

- AI may be used for architectural guidance, code reviews, and writing test cases.
- The implementation of the systems themselves is intentionally human-written.
- The point is to understand the requirements personally and turn them into code directly, rather than outsourcing the implementation step.
